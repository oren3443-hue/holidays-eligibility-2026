# חישוב זכאות פסח 2026

אתר סטטי לחישוב זכאות עובדים לחגי פסח (פסח א, חול המועד, פסח ב). כל החישוב בצד הלקוח – ללא שרת, ללא העלאה לאף מקום.

## שימוש

1. פתח את `index.html` בדפדפן (או היכנס לאתר אחרי פריסה ל־GitHub Pages).
2. העלה את 5 קבצי האקסל:
   - **פרטי עובדים** (מיכפל)
   - **נוכחות 3 חודשים הקודמים** (סטיגו)
   - **נוכחות אפריל** (סטיגו) – פילטר רלוונטיות
   - **נוכחות 31.3** – לזכאות פסח א
   - **נוכחות 9.4** – לזכאות פסח ב
3. לחץ "חשב זכאות" – הטבלה מתעדכנת.
4. לחץ "הורד אקסל" – מקבל קובץ פלט עם כל העמודות (גלם + מחושב).

## לוגיקת חישוב

### תאריכים – פסח 2026
- 31.3 (ג') – יום בדיקת זכאות לפסח א
- 1.4  (ד') – ערב פסח, יום חופש
- 2.4  (ה') – פסח א
- 3.4  (ו') – חול המועד (חופש לעובד 6 ימים)
- 5.4–7.4 – חול המועד (חופש לכולם)
- 8.4  (ד') – פסח ב
- 9.4  (ה') – יום בדיקת זכאות לפסח ב

### זכאות
- ותק: התחלת עבודה ≤ 1.1.2026.
- פסח א: משמרת/חופש/מחלה/מילואים ב־31.3.
- פסח ב: משמרת/חופש/מחלה/מילואים ב־9.4.

### חול המועד – ימי חופש מהיתרה
- עובד 5 ימים בשבוע: עד 4 ימים (1.4 / 5.4 / 6.4 / 7.4).
- עובד 6 ימים בשבוע: עד 5 ימים (1.4 / 3.4 / 5.4 / 6.4 / 7.4).
- כמות בפועל = `min( floor(יתרה), תקרה )`.
- ניצול בסדר כרונולוגי, מהמוקדם למאוחר.
- שעות ליום = ממוצע (שעות עבודה / ימי משמרת בפועל) ב־1.1–31.3.
- תקרת שעות:
  - 5 ימים: 8.4 שעות בכל יום.
  - 6 ימים: 8 ביום חול, 6 בערב חג ובשישי (1.4, 3.4, 7.4).

## פרטיות

הקבצים נשארים במחשב שלך. כל הקריאה והחישוב מתבצעים בדפדפן באמצעות
[SheetJS](https://sheetjs.com/). ה־`.gitignore` חוסם הוספת קבצי אקסל ל־repo.

## פריסה ל־GitHub Pages

```sh
git init
git add .
git commit -m "initial: Pesach eligibility calculator"
gh repo create --public --source=. --remote=origin --push
gh api repos/{owner}/{repo}/pages -X POST -f source.branch=main -f source.path=/
```

(או דרך ממשק GitHub: Settings → Pages → Branch: main / root.)
